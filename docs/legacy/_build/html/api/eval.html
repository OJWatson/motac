This is intentionally tiny (toy-data sized): it supports M5 by providing a single helper that wires together fit -> forecast -> scoring.

Parameters:

  • kernel (ndarray) – Discrete lag kernel used by the model.

  • y (ndarray) – Count matrix (n_cells, n_steps).

  • n_train (int) – Number of steps to use for training.

  • horizon (int) – Number of held-out steps to score.

Return type:

BacktestResult with mean negative log likelihood on the held-out window.

motac.eval.evaluate_synthetic(config)[source]

Run a tiny, deterministic end-to-end backtest on synthetic data.

Steps

  1. Generate a random world and simulate counts for train+horizon steps.

  2. Fit (mu, alpha, beta) by MLE on the training window.

  3. Produce horizon-step predictive samples and predictive summaries.

  4. Compute a minimal metric set (held-out NLL, RMSE/MAE, quantile coverage).

returns:

  • config: dict

  • fit: dict

  • forecasts: dict

  • metrics: dict

rtype:

dict with keys
